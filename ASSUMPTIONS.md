# ASSUMPTIONS — 前提と既定値の記録

情報が不足した箇所は、作業を止めずに合理的な既定値を採用しました。記録します。

## プラットフォーム / 移行
- 当初の Godot 指示は、後続の最重要変更（Web/PWA, TypeScript, Phaser 4.2.0）により**上書き**された
  ものと解釈。既存 Godot 実装は無かったため、Web 版を新規構築した。
- GitHub Pages のプロジェクトサイトを想定し、`base` の既定を `/dq/`（リポジトリ名 `dq`）とした。
  ルートドメイン等では `BASE_PATH=/` でビルド。

## データ形式
- マスターデータは JSON ではなく **TypeScript 定義**を採用（strict型での静的検査・enum共有のため）。
  データ駆動性は `DataRegistry` 経由参照＋`validateData()` で担保（理由は DESIGN.md §2）。

## バランス・数値（すべて独自設定。原作の数値・経験値表・配合表は不使用）
- 最大レベル50。経験値: `expToNext(L)=round(6L²+10L+12)`。撃破経験値: `round(8+lv²*1.5)`。
- 能力成長: `base*(1+growth*(L-1))`（growth は種族ごと 0.04〜0.12 程度の小数）。
- ダメージ/回復/会心/毒の式は DESIGN.md §3 の通り（会心6%/×1.5、毒=最大HP8%/ターン）。
- 属性相性は独自の循環（火>風>地>水>火、光⇔影）。倍率 0.0/0.5/0.75/0.8/1.0/1.25/1.5。
- 共鳴成功率・配合補正上限(+12/ステ)・保有上限(50) は本書/DESIGN記載の独自式。

## 戦闘仕様の判断
- 戦闘不能だった味方の取得経験値は生存者の50%（"一定割合"の解釈）。経験値はパーティ人数で分配。
- 2回行動などの追加行動は**1ターン1回まで**（無限ループ防止）。防御は被ダメ0.5倍・ターン終了まで。
- 直接命令以外の作戦の味方は、敵AIと共有の `ActionEvaluator` で自動行動（設定値=作戦の重み）。
- ボス戦は逃走不可・加入不可。エリア補正は既定0。

## スターター / 進行
- 新規開始時のスターターは Lv5 の `mossfang / emberwisp / gravelkin`（獣・精霊・岩石）3体。
  所持金100G、初期道具（癒し草×3・泉の雫×2・澄み葉×1・共鳴の鈴×1）。
- クリア条件はボス `dirgewarden`（番人の間, 洞窟最奥）の撃破1つ。番人の間はいつでも挑戦可能
  （推奨レベルは探索で育成して到達する想定）。

## 探索
- マップは本格的なタイルマップではなく、タップ移動＋「探索を進める」ボタンによる軽量エンカウント方式
  （タッチ操作の確実性とMVPのループ完成を優先）。遭遇率: 移動0.45 / 探索ボタン0.70。

## 音・演出
- 効果音は WebAudio 合成音（音声ファイル無し＝欠落しても停止しない）。BGM 楽曲は未同梱。
- グラフィックは円＋絵文字グリフ＋単色矩形のプレースホルダー（商用素材不使用）。
  絵文字グリフは端末フォントに依存（表示が無くてもゲーム進行に影響なし）。

## 既定の操作
- キーボード: 方向/WASD=移動, Enter/Space=決定, Esc/Backspace=戻る。ゲームパッドは Phaser の
  既定有効化のみ（明示的マッピングは最小）。

## 検証環境の制約
- 本作業環境ではブラウザを起動できず（Playwright のブラウザ取得がネットワーク制限で失敗）、
  画面の目視・E2E は未実行。型チェック・lint・52ユニットテスト・本番ビルドで検証した（TASKS.md 参照）。
